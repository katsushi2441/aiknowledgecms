## Beat 1 — Thesis / Hook (0.0-7.2s)

**VO:** AIKnowledgeCMS is not a normal admin panel. It is a CMS operated by an agent loop.

**Concept:** The website hero becomes a living product card. We enter on a bright dotted canvas, then the phrase “agent loop” lifts out of the page as the product thesis.

**Visual:** Full-frame white canvas with faint dot grid. Large floating screenshot card of the hero section tilted slightly. Foreground title rebuilds in oversized type: “Not an admin panel” then “An agent loop.” Orynth badge floats top-left as credibility proof. Purple-to-cyan ring traces an infinity loop behind the screenshot.

**Motion:** Screenshot slides in from the right and slowly zooms. Headline words cascade with mixed x/y/scale entrances. SVG loop path draws behind the card. Badge floats with tiny pulse.

## Beat 2 — The Growth Loop (7.2-15.2s)

**VO:** Every tick senses the site, researches sources, triages issues, creates content, verifies quality, and distributes the result.

**Concept:** The loop becomes an operating cycle, not a list. Stage cards orbit around a central “tick” core.

**Visual:** Six stage pills: SENSE, RESEARCH, TRIAGE, ACT, CREATE, DISTRIBUTE. Behind them, the captured loop-stage screenshot scrolls gently inside a browser frame. A green MEASURE arrow closes the loop.

**Motion:** Stage pills pop in clockwise. Connecting lines draw between them. Screenshot performs a vertical scroll reveal. The central tick counter pulses once per phrase.

## Beat 3 — Loopfile / Guarded Automation (15.2-24.2s)

**VO:** The loop is declared in one Loopfile: KPIs, cadence, budgets, gates, escalation, and adapters. That means growth is not a manual task. It becomes a measurable, auditable operating cycle.

**Concept:** The technical proof. We zoom into the loopfile code window and turn its lines into product capabilities.

**Visual:** Dark code window from the site sits left in a white studio scene. On the right, four large cards: KPIs, Budgets, Gates, Adapters. Thin purple lines connect code lines to cards. A small SQLite memory ledger strip moves at the bottom.

**Motion:** Code window enters with a soft snap. Highlight bars sweep over loopfile lines. Capability cards slide in from alternating directions. Ledger dots count up.

## Beat 4 — Proof / CTA (24.2-34.8s)

**VO:** AIKnowledgeCMS is already dogfooding itself: live loop reports, generated articles, and gated publishing are running on the reference site. There is no admin panel. There is a loop.

**Concept:** Proof, not promise. The reference site screens become an evidence wall, then resolve into the product URL.

**Visual:** Three stacked cards: live loop reports, generated articles, reference deployment. Captured screenshots from guardrails and roadmap sections overlap with subtle depth. Final frame: AIKnowledgeCMS wordmark, URL, GitHub/Orynth CTA.

**Motion:** Cards fan in like audit records. Green checkmarks stamp “running” and “gated.” The final line locks into place with a clean purple sweep.
