# Design System

## Overview

AIKnowledgeCMS uses a clean white-studio product identity with a technical but approachable tone. The page is built from spacious centered sections, rounded cards, fine dotted patterns, thin borders, and strong typographic hierarchy. Purple and cyan gradients mark the core idea: a CMS operated by an agent loop rather than a manual admin panel. The video should keep the bright canvas, crisp cards, and loop-engineering vocabulary while scaling all elements for motion.

## Colors

- **Ink**: `#101828` — primary headlines, buttons, strong labels.
- **Body**: `#475467` — explanatory body text.
- **Faint**: `#98A2B3` — metadata, small labels.
- **White Surface**: `#FFFFFF` — primary background and cards.
- **Soft Surface**: `#F7F9FC` — section contrast and large panels.
- **Line**: `#E9EDF3` — card borders and dividers.
- **Line Strong**: `#D0D5DD` — button borders and stronger rules.
- **Accent Purple**: `#4F46E5` — primary brand accent and loop labels.
- **Cyan**: `#0891B2` — secondary gradient endpoint.
- **Green**: `#059669` — success, feedback, measure loop.
- **Amber**: `#B45309` — phase/status pill.
- **Code Surface**: `#0F172A` — loopfile/code window.
- **Code Bar**: `#1E293B` — code window chrome.

## Typography

- **Primary Sans**: Inter, weights 400, 700, 800, 900. Used for headings, product copy, labels and UI.
- **Monospace**: SFMono-Regular, weights 400, 700. Used for loopfile snippets, stage labels, and technical metadata.
- **Video hierarchy**: headlines 72-112px, body 28-38px, labels 20-24px, code 24-30px.
- **Voice split**: Inter for product statements; SFMono-Regular for loop stages, telemetry and execution details.

## Elevation

Depth comes from white cards over a pale canvas, thin borders, soft shadows, and faint radial blue/purple glows. The site avoids heavy dark surfaces except for the loopfile code window, which should appear as a controlled technical insert rather than the dominant style.

## Components

- **Hero Dotted Canvas**: light background with small dot grid and centered product thesis.
- **Loop Stage Cards**: numbered cards for SENSE, RESEARCH, TRIAGE, ACT, CREATE and DISTRIBUTE.
- **Gradient Wordmark Accent**: “agent loop” uses purple-to-cyan emphasis.
- **Loopfile Code Window**: dark terminal-like panel showing declarative configuration.
- **Guardrail Cards**: budget, dry-run, kill switch and escalation cards.
- **Dogfooding Panel**: soft gradient reference deployment card.
- **Orynth Badge**: small proof badge in the header.

## Do's and Don'ts

### Do's

- Keep a bright white/soft-gray canvas with visible purple/cyan accents.
- Use large, confident typography and generous product-video spacing.
- Present screenshots as floating cards with motion, not flat still images.
- Show the loop stages as an operating system, not a blog layout.
- Use code-window styling only for loopfile moments.

### Don'ts

- Do not use black full-screen backgrounds.
- Do not replace the product with generic AI stock imagery.
- Do not make the page screenshots static; pan, crop, tilt or layer them.
- Do not add fake dashboard claims not present on the site.
- Do not use unreadably tiny web text in video frames.
