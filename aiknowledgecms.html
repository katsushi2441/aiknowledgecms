Loop Engineering for Website Growth

人が運用するCMSではなく、
エージェントループが運用するCMS。

AIエージェントが知識サイトの上で成長ループを回し続けます—— サイトの状態を監視し、必要な情報を集め、課題を整理し、課題に対応し、 コンテンツを生成・配信し、その結果を次の周回に反映する。 管理画面の代わりに、ループがあります。

The Growth Loop

1周（tick）＝6ステージ

暴走する while(true) ではなく、再開可能な「tick」の繰り返し。 スケジューラがtickを起動し、エージェントは手順書（playbook）に従ってステージを進み、結果はすべて計測されて次に引き継がれます。

1SENSE

サイトを監視する

アクセス解析・Search Console・アクセスログ・収益・死活を、ループのメモリに構造化された観測値として取り込みます。

2RESEARCH

情報を収集する

トレンド・情報源・競合のシグナルをコレクタアダプタが収集。意思決定の材料になります。

3TRIAGE

課題を整理する

観測値を優先度つきの課題キューに変換。PV下落、インデックスエラー、コンテンツの穴、古くなったページ。

4ACT

課題に対応する

エージェントがplaybookに従って課題を処理。サイト変更はgit worktreeを経由し、プルリクエストとして反映します。

5CREATE

コンテンツを生成する

記事・動画・シェアカードを生成。ただし別の検証エージェントによる品質ゲートを通らない限り、何も公開されません。

6DISTRIBUTE

配信・告知する

パブリッシャ／アナウンサアダプタが、サイト・SNS・動画プラットフォームへ届けます。

↺ MEASURE — 結果はSENSEに書き戻される。何がサイトを成長させたかをループが学び、次はそれを増やす。
Architecture

6つの構成要素、エージェント非依存

エージェントは交換可能。任意のagent CLI（claude -p / codex exec）がステージ実行器になります。 プロダクトの本体はループです。

⏱

Tickとスケジューリング

cron/systemdがケイデンスに従って再開可能なtickを起動——監視は毎時、生成は毎日、対応は毎週。状態は永続化され、クラッシュしても安全に再開できます。

📖

Playbook（手順書）

手順書はファーストクラスの成果物。エージェントはステージ実行前に必ず手順書を読みます——本番での即興を許しません。

🔌

アダプタ

センサ／コレクタ／ジェネレータ／パブリッシャ／アナウンサを差し替え可能に。Search ConsoleをPlausibleに、FTPをGitHub Pagesに。

🛡

品質ゲート

生成者と検証者は別エージェント。事実性・SEOのゲートに落ちたコンテンツは修復または破棄され、黙って公開されることはありません。

🗄

メモリ

SQLiteに課題キュー・コンテンツ台帳・メトリクス時系列・意思決定ログを保持。すべて監査可能です。

🌿

Worktree

サイトを変更するアクションは隔離されたgit worktreeで実行し、レビュー可能なプルリクエストとして着地します。

loopfile.yaml
site: aiknowledgecms.exbridge.jp
kpi: [pv_weekly, affiliate_clicks, indexed_pages]
cadence:
  sense: hourly
  create: daily
  act: weekly
budgets:
  articles_per_day: 3
  llm_cost_per_tick_usd: 2.0
gates: [factuality_check, seo_check]
escalate_when: [revenue_drop_30pct, index_errors]
adapters:
  sensors: [gsc, simpletrack]
  publisher: ftp_heteml
  announcer: aixsns
Declarative

サイトごとに1枚の「Loopfile」

KPI・ケイデンス・予算・ゲート・エスカレーション——ループ全体を1つのファイルで宣言します。ランナーにLoopfileを渡せば、サイトが育ち始めます。

  • KPIが「このサイトにとっての成長」を定義し、ループはそれを最適化する
  • ケイデンスでステージごとの実行頻度を独立に制御
  • 予算とゲートはランナーが強制する——エージェントの判断任せにしない
Loop Engineering

ループは判断を増幅する——
だからブレーキは仕様の一部

ガードレールのない自律無限ループは資産ではなく負債です。後付けではなく、最初から組み込みます。

💰

予算上限

tickあたりのLLMコスト・記事数・API呼び出しに絶対上限。財布より先にループが止まります。

🧪

Dry-run

全ステージが副作用ゼロで実行可能。推論の中身を監査してから信頼できます。

🛑

Kill switch

フラグ1つでループ全停止。状態は永続化されているので再開も安全です。

🙋

エスカレーション

宣言した条件で人間に通知（メール／LINE）。検証は人間の責任のままです。

Dogfooding

このサイト自身が参照実装

aiknowledgecms.exbridge.jp は、このフレームワークの最初のインスタンスとして再構築中です。 AIエージェント経済をテーマにした実証メディアを、ループ自身が記事・動画・シェアカード・収益まで 生成・計測し、毎tickのレポートを公開します。フレームワークの証明は、あなたが今読んでいるこのサイトです。

Roadmap

これからの進み方

P0

構想と仕様 ✓

マニフェスト、Loopfile仕様、アーキテクチャ。初日から公開。

P1

コアtickランナー + SENSE / TRIAGE / REPORT ✓

実センサが課題キューをSQLiteに蓄積し、毎時のtickが公開ループレポートを出力——このサイトで稼働中。

P2

CREATE / DISTRIBUTE + 品質ゲートいまここ

ループが素材を収集し、記事を生成・公開。独立した検証エージェントのゲートを通過したものだけが世に出ます——不合格ドラフトは理由つきで台帳に残ります。

P3

ACT + シェア機能

worktree経由でサイト自体を修正するplaybook。診断カード・成長カード・ダッシュボードなど、シェアしたくなる機能。
